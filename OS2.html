关键点：
  • CPU同一时间只能处理一个任务。
  • 当一个任务去做 I/O 时，CPU 会空闲出来给另一个任务使用（多道批处理的核心）。
  • I/O 设备通常也视为串行资源。

问题 (2) - 空闲等待

---

解析: P1 在 155ms 结束，此时 P2 还在做 I/O (直到 175ms)。这中间 CPU 无事可做。

问题 (2) - 利用率

CPU 利用率为多少？

---
CPU 忙碌时间 / 总时间

计算: (210 - 20) / 210 ≈ 90.5%

P1 的生命周期

  • 0-40ms: CPU计算 (40ms) - P1先到先得
  • 40-100ms: I/O操作 (60ms) - 释放CPU给P2
  • 100-130ms: 等待CPU - P2占用CPU中
  • 130-155ms: CPU计算 (25ms) - P2去I/O了，CPU空闲
  • 155ms: P1 结束

P2 的生命周期

  • 0-8ms: 未到达
  • 8-40ms: 等待CPU - P1占用CPU中
  • 40-130ms: CPU计算 (90ms) - P1去做I/O了，拿到CPU
  • 130-175ms: I/O操作 (45ms) - 释放CPU给P1
  • 175-210ms: CPU计算 (35ms) - P1已结束，CPU空闲
  • 210ms: P2 结束
